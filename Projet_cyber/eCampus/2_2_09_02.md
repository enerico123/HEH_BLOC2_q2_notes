# Réponses 

## 2.2.1 Expliquez la notion de pare-feu avec état (stateful firewall).

Statefull firewall : garde en mémoire l'état des connexions, si une connexion à déjà été établie il va la mettre en mémoire pour directement l'accepter la prochaine fois  

## 2.2.2 Citez trois moyens de renforcer la sécurité liée à l'accès administratif d'un FortiGate.

3 Moyen : 
- A2F 
- White List d'adresse IP 
- désactiver les accès inutiles 
- changer le port par défaut 
- utiliser des comptes nominatif 
- log + alerts 

## 2.2.3 Citez 5 fonctionnalités de sécurité (profils de sécurité) supportées par un FortiGate/NGFW.

- Antivirus 
- IPS (détecte et bloque les attaques réseau)
- Web Filtering (bloques certaines catégories de sites)
- Application Controle (Contrôle les app)
- Anti-spam 

## 2.2.4 Comparez les profils d’administrateur super_admin et prof_admin.

super_admin : a les droits sur absolument tout 

prof_admin : accès restreint (peut gérer politiques et profils) super_admin dédié à un seul VDOM 

## 2.2.5 Citez les méthodes qui peuvent être utilisées pour délivrer un token à un utilisateur lors de l'authentification à deux facteurs.

- token matériel (clé physique)
- App mobile (FortiToken hardware)
- SMS 
- email 

## 2.2.6 Citez les deux modes de fonctionnement (Modes of operation) d’un FortiGate. Quel est le plus couramment utilisé ?

- Mode NAT (Rooted Mode) 

- Mode Transparent (Bridge Mode) 

## 2.2.7 Citez les caractéristiques principales des deux modes de fonctionnement (Modes of operation) d’un FortiGate.

- Mode NAT : agit comme un routeur + filtre les paquet  

- Mode Transparent : fait juste passer le trafic en le filtrant
